# Docs Archive

Historical implementation records for Yuval Fit OS. These documents captured a
plan or a state that was true at the time they were written and has since been
**superseded or removed from the active flow**. They are kept so project history
and the original reasoning are not lost.

## How to read these

- **[`PROJECT_STATE.md`](../PROJECT_STATE.md) is the current source of truth.**
  Always read it first for what the app is and how it is wired today.
- Documents in this folder **may be outdated** and **must not override**
  `PROJECT_STATE.md` or the active feature docs in `docs/`.
- Do **not** treat anything here as current requirements without confirming
  against `PROJECT_STATE.md` and the relevant active doc.
- Nothing here is deleted — moving over deleting keeps the history intact.
